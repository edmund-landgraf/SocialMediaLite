How to setup on your own server

Run SocialMediaLite from the monorepo on your laptop, homelab, or VPS. These steps assume the repository is cloned and you work from the repo root.

1. Prerequisites

  • Node.js 20 or newer and npm
  • PostgreSQL 14+ (Docker, WSL, or native install)
  • Git, if you are cloning the repository
  • Optional: Meta Facebook app for real Facebook login and timeline import

2. Get the code

git clone <your-repo-url> SocialMediaLite
cd SocialMediaLite

3. Database

With Docker Compose from the repo root:

docker compose up -d

Or use your own Postgres instance. Create a database and user, then note the connection string for DATABASE_URL.

4. Environment file

  1. Copy .env.example to .env at the repo root.
  2. DATABASE_URL — e.g. postgresql://USER:PASSWORD@localhost:5432/socialmedialite?schema=public
  3. SESSION_SECRET — long random string (required in production).
  4. WEB_ORIGIN — exact browser origin users will use (dev: http://localhost:5174 if Vite moved off 5173).
  5. API_PORT — default 3001.
  6. STORAGE_PROVIDER=local and STORAGE_LOCAL_ROOT for uploaded images (see .env.example).

Facebook Login (optional)

Set FACEBOOK_APP_ID, FACEBOOK_APP_SECRET, and FACEBOOK_REDIRECT_URI. The redirect must be your public web origin plus /api/auth/facebook/callback — not the API port alone.

5. Install dependencies

npm install

postinstall builds @socialmedialite/shared and runs prisma generate.

6. Apply database migrations

npm run db:deploy

Run again after pulling changes that include new Prisma migrations.

7. Development (local)

API and web together:

npm run dev

Or separate terminals:

npm run dev -w apps/api
npm run dev -w apps/web
  • Web — URL printed by Vite (often port 5173 or 5174)
  • API — http://localhost:3001; Vite proxies /api and /assets

8. Production build (overview)

Production typically uses npm run build, serves the web dist/ with nginx, and runs the API with PM2. See scripts/deploy-prod.sh and ecosystem.config.cjs in the repo for your environment. Set NODE_ENV=production, a strong SESSION_SECRET, and WEB_ORIGIN to your HTTPS site URL.

Troubleshooting

  • P2022 / column does not exist — run npm run db:deploy; restart the API.
  • Cannot find module @socialmedialite/shared — npm run build -w @socialmedialite/shared
  • Facebook OAuth redirect mismatch — FACEBOOK_REDIRECT_URI must match the host and port in the browser bar.
  • Port already in use — stop the old Node/Vite process or use the new port Vite prints.
  • ECONNREFUSED to Postgres — confirm Postgres is running and DATABASE_URL is correct.
